Deploying Pure Static Sites

In this lab, we are taking Path 1. We will use generative AI to write a beautiful API documentation page, and then deploy it globally using GitHub Pages.

The Goal

To understand how files move from your local computer to a global Content Delivery Network (CDN) with zero cost and zero servers to manage.

The Constraint

GitHub Pages only hosts static files. No Python, no Node.js, no databases. If your site needs to compute logic securely, this is the wrong path.

Step 1

Generating Technical Documentation

Now, we will use AI to analyze our existing application and generate a professional documentation page based on the actual code.

Prompt: Code Analysis & Docs
analyze index.html, app.js, style.css and create a github pages documentation. 1. Create a single-file static HTML documentation page for an "Idea Tracker API" and include all endpoints found in the code. 2. Create a folder named "docs" in the current directory. 3. Create a file named "index.html" inside the "docs" folder and paste your generated code inside it. Requirements: 1. Use Tailwind CSS via CDN. 2. Include a sidebar navigation menu. 3. The main content should have sections for: Introduction, Authentication, and all API Endpoints in index.html, app.js, etc. 4. Use a dark-mode theme (slate-900 background) with neon blue accents. 5. Provide code blocks showing example JSON responses. 6. Ensure the generated HTML is valid, semantic, and standards-compliant.

Action Required

1. Paste the prompt above into your AI assistant along with the contents of your index.html, app.js, and style.css.

2. Review the AI's response and follow its instructions to finalize your docs/index.html documentation site.

Step 2

Publishing via AI

Instead of running Git commands manually, we'll ask the AI to handle the deployment preparation and version control for us.

1

Create a New Repo

Go to github.com/new and create a repository named idea-api-docs. Keep it Public.

2

Use the AI Publisher

Prompt 2: Git Publishing
I want you to publish my current changes to the GitHub repository. Please perform the following: 1. Create a .gitignore file suitable for a standard web project. 2. Create a README.md file with a brief description of the "Idea Tracker API" project. 3. Add all files, commit them with the message "Initial deployment of API docs", and push them to the main branch. Please ask me for the GitHub Repository URL before proceeding, and resolve any issues or conflicts that may arise during this process.

Pro Tip

The AI can help you write perfect commit messages and ensure you don't accidentally upload sensitive files like API keys or local logs by creating the .gitignore for you!

Step 3

Configuring GitHub Pages

Your code is on GitHub. We need to tell GitHub to serve the content specifically from the docs folder.

  1. 1. Go to Settings

    On your repository page in GitHub, click the "Settings" tab at the top right.

  2. 2. Navigate to Pages

    In the left-hand sidebar, scroll down to the "Code and automation" section and click on "Pages".

  3. 3. Select the Source & Folder

    Under "Build and deployment", change "Branch" to main. In the folder dropdown next to it, select /docs.

  4. 4. Save & Deploy

    Click the "Save" button. GitHub Actions will now build your site from the /docs folder.

Step 4

Verification & Troubleshooting

It takes about 1-2 minutes for GitHub to distribute your file globally. Let's verify the deployment.

Your Site is Live

Refresh your GitHub Pages settings screen. You should see a notification at the top containing your live URL.

https://YOUR_USERNAME.github.io/idea-api-docs/

Troubleshooting 404 Errors

If you see a 404 page, check these common causes:

  • Ensure your main HTML file is named exactly index.html (lowercase).
  • Ensure index.html is inside the docs folder.
  • Ensure you selected the /docs folder in GitHub Pages settings.

Deployment Complete

You successfully used AI to generate documentation based on your code and deployed it to GitHub Pages via the /docs folder.
